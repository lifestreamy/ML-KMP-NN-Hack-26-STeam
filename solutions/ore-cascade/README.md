# Итоги работы

## Web-demo. Запуск - [🚀 Live Demo](https://ore-cascade-gin79umiu9ojvq5bdhttxn.streamlit.app/)

Итак, что мы в итоге получили.

Мы решали классическую для геологии проблему: разметка руды по трём классам — **оталькованная**, **рядовая**, **труднообогатимая** — сейчас делается вручную, глазами эксперта под микроскопом. Это медленно, зависит от усталости и опыта конкретного человека, и на больших объёмах проб просто не масштабируется.

Мы построили **двухступенчатый каскад**, а не одну большую модель — и это осознанное решение.

## Stage 1 — классическое компьютерное зрение

Первая ступень — KMeans-кластеризация в LAB-пространстве плюс анализ локальной текстуры. Она отделяет тальк от всего остального **без единого обучаемого веса** — а значит, полностью объяснима: для любого предсказания мы можем показать геологу конкретную зону на снимке и сказать, почему модель так решила. Это критично для доверия в промышленной задаче.

> **Точность на этом этапе — 92%**

## Stage 2 — нейросеть

Вторая ступень включается только тогда, когда тальк не найден в достаточном объеме, и решает более тонкую задачу — отличить рядовую руду от труднообогатимой. Здесь зависимости более сложныеи в ход пошла "простая" быстрая нейросетевая модель CNN, с 40 эпохами обучения.

После калибровки порога решения по f1-метрике:

| Метрика | До калибровки | После калибровки |
|---|---|---|
| Accuracy | 90% | **95%** |
| Recall (труднообогатимая) | 87% | **99%** |

Это значит, что мы почти не пропускаем сложную руду — а именно её пропуск дороже всего обходится на обогатительной фабрике.

## Главное

У нас уже есть не просто ноутбук с экспериментом, а **рабочее приложение**: каскад обёрнут в Streamlit с загрузкой снимков, визуализацией зон и экспортом PDF-отчёта.

## **Это не идеальный инструмент с большим набором функций, но это прототип, которым геолог может пользоваться уже сейчас.**

## Установка и запуск

```bash
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # Linux/Mac

pip install -r requirements.txt

streamlit run streamlit_app.py
```

Откроется браузер на `http://localhost:8501`.

### AMD GPU (Windows)

Раскомментируйте `torch-directml` в `requirements.txt` перед установкой —
тогда Stage 2 (CNN) будет использовать GPU. Stage 1 всегда работает на CPU
независимо от этого (KMeans/OpenCV).

## Возможности приложения

- **Вкладка «Одно изображение»** — загрузка одного файла (или встроенный
  пример), запуск каскада, вывод финальной метки, диагностических
  процентов визуализация зон (сульфиды/тальк) и density-карты, скачивание результата
  в JSON.
- **Вкладка «Пакетная обработка»** — загрузка сразу нескольких файлов,
  таблица результатов, гистограмма по классам, скачивание CSV.

## Известные ограничения

Такие же, как у исходного ноутбука:

- Stage 1 выполняется на CPU в любом случае — на очень крупных
  изображениях один файл может обрабатываться несколько секунд.
- При сильном отличии новых
  данных (другое освещение/микроскоп) может понадобиться повторная
  калибровка.
