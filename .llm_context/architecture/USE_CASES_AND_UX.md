## Архитектурное ограничение: Active Learning
Тренировка (дообучение YOLO/SAM) на лету НЕ запускается. Система только собирает датасет.

---

## UC-1: Очередь и Пакетная обработка (Основной флоу)
**User Flow:**
1. Геолог выбирает пачку файлов.
2. Нажимает "Анализировать".
3. **Guardrail:** Ktor Gateway проверяет доступность Python ML. Если ML недоступен, клиент мгновенно получает ошибку 503, и задачи НЕ встают в очередь. В UI появляется красный баннер с ошибкой.
4. Если ML доступен, UI отправляет файлы. Ktor отвечает `200 OK` с массивом UUID задач.
5. Задачи обрабатываются (FIFO). Статусы обновляются через SSE.
6. Готовая карточка кликабельна: открывает Canvas с масками.

## Индикация состояния системы (Traffic Lights UX)
В правом верхнем углу интерфейса постоянно отображаются два светофора:
- **Ktor Gateway**: Зеленый (доступен) / Красный (отключен).
- **Python ML**: Зеленый (доступен) / Красный (отключен).
*UX-правило:* Если связь с Ktor теряется (красный свет), все задачи в статусе `В очереди` или `Обработка` автоматически переводятся в статус `Ошибка: Соединение потеряно`, чтобы не блокировать интерфейс бесконечным ожиданием.

## UC-2: Экспертная разметка (Сбор датасета)
1. На экране результатов геолог видит неверную разметку.
2. Выделяет полигоном область, указывает класс.
3. Сохраняет правки (в `/dataset/feedback` как пару TIFF+JSON).
