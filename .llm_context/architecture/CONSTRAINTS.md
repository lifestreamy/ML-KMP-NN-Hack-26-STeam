# CONSTRAINTS

> Жёсткие ограничения от организаторов. Всё, что ниже — не обсуждается.

## Дедлайны
- `2 июля 22:00` — публикация полных условий задач
- `4 июля 23:59` — дедлайн загрузки решения (строго)
- `5 июля 08:00–12:00` — проверка решений жюри
- `5 июля 13:00–16:00` — защиты финалистов

## Формат сдачи (обязательно всё)
1. Ссылка на репозиторий (GitHub / GitLab / Mercury)
2. Яндекс/Google Диск:
   - Архив с исходным кодом
   - Видео-демо ≤ 5 минут
3. Ссылка на презентацию (.pptx / .pdf / YouNote / Figma)
4. Ссылка на развёрнутое решение (желательно, не обязательно)

## Технические требования задачи
- Входные данные: OM-изображения (TIFF, PNG, JPEG), только оптическая микроскопия
- Производительность: ≤ 5 минут на одно панорамное изображение (до 10 000×10 000 px) на CPU/GPU
- Точность классификации срастаний: F1-score ≥ 90%
- Ошибка оценки доли талька: ≤ ±3% от экспертной разметки
- Локальное развёртывание: обязательна поддержка (конфиденциальность геологических данных)
- Интерфейс: понятен геологу без знания ML

## Что НЕ нужно (убрали из условий)
- SEM-изображения — нет
- XRD-профили — нет
- Интеграция с ГИС — опционально
